## 🎬 演示流程（建议顺序）

### 第一部分：系统介绍（2分钟）

**亮点说明：**
"这是一个面向患者和医生双端的智能医疗语音助手系统。"

**核心功能：**
1. **患者端**：智能导诊、症状咨询
2. **医生端**：辅助诊断、用药查询
3. **基础能力**：语音识别、情感识别、声纹识别

**技术栈：**
- 语音识别：Paraformer-Large
- 对话生成：Qwen2.5-0.5B
- 知识检索：RAG + 医疗知识库

---

### 第二部分：患者端演示（3分钟）

#### 演示1：智能导诊

**场景**：患者不知道该挂哪个科

**使用curl演示：**
```bash
curl -X POST http://localhost:6007/patient/triage \
  -H "Content-Type: application/json" \
  -d '{
    "symptoms": ["发烧", "咳嗽", "头痛"],
    "age": 8,
    "gender": "男"
  }' | python -m json.tool
```

**预期结果：**
```json
{
  "status": "success",
  "result": {
    "recommended_department": {
      "name": "呼吸内科"  // 或儿科
    },
    "priority": "normal",
    "reason": "根据您的症状（发烧、咳嗽），推荐就诊科室"
  }
}
```

**讲解要点：**
- ✓ 系统自动识别了儿童（8岁）
- ✓ 根据呼吸道症状推荐科室
- ✓ 评估了紧急程度（normal）

#### 演示2：紧急情况识别

```bash
curl -X POST http://localhost:6007/patient/triage \
  -H "Content-Type: application/json" \
  -d '{
    "symptoms": ["剧烈胸痛", "呼吸困难"],
    "age": 55,
    "severity": "emergency"
```

**预期结果：**
```json
{
  "recommended_department": {
    "name": "急诊科"
  },
  "priority": "emergency",
```

**讲解要点：**
- ✓ 系统识别出危急症状

- ✓ 标记为紧急优先级

### 第三部分：医生端演示（4分钟）

#### 演示3：症状分析与疾病推断

**场景**：医生输入患者症状，系统辅助诊断

```bash
curl -X POST http://localhost:6007/doctor/analyze-symptoms \
  -H "Content-Type: application/json" \
  -d '{
    "symptoms": ["高热", "头痛", "肌肉酸痛", "乏力", "咳嗽"],
    "patient_info": {
      "age": 35,
      "gender": "男"
    }
  }' | python -m json.tool
```

**预期结果：**
```json
{
  "status": "success",
  "analysis": {
    "possible_diseases": [
      {
        "name": "流行性感冒",
        "confidence": 85.5,
        "matched_symptoms": ["高热", "头痛", "肌肉酸痛", "乏力"],
        "severity": "moderate",
        "treatment": "抗病毒药物、对症治疗"
      }
    ],
    "confidence": "high",
    "suggestions": [
      "症状高度符合流行性感冒特征",
      "建议检查项目以确诊"
    ],
    "additional_questions": [
      "症状是什么时候开始的？",
      "有无既往病史或家族史？"
    ]
  }
}
```

**讲解要点：**
- ✓ 给出可能疾病及置信度（85%）
- ✓ 列出匹配的症状
- ✓ 提供诊断建议
- ✓ 生成追问问题

#### 演示4：药物查询

```bash
curl -X POST http://localhost:6007/medication/query \
  -H "Content-Type: application/json" \
  -d '{"medication": "阿莫西林"}' | python -m json.tool
```

**预期结果：**
```json
{
  "status": "success",
  "medication": {
    "name": "阿莫西林",
    "category": "抗生素（青霉素类）",
    "indications": ["呼吸道感染", "泌尿道感染"],
    "dosage": {
      "adult": "0.5g，每8小时一次",
      "child": "根据体重计算，20-40mg/kg/日"
    },
    "contraindications": ["青霉素过敏者禁用"],
    "side_effects": ["恶心", "呕吐", "腹泻"]
  }
}
```

**讲解要点：**
- ✓ 完整的药品信息
- ✓ 成人和儿童剂量
- ✓ 禁忌症和副作用

#### 演示5：药物相互作用检查（重点）

## ⚡ 应对可能的提问

### Q1: "你的数据从哪里来？"
**A:** "医疗知识来自权威医学教材和临床指南，药品信息参考国家药品监督管理局发布的说明书。实际应用需要接入更完整的医疗数据库。"

### Q2: "准确率如何？"
**A:** "导诊模块基于症状-科室映射规则，准确率在85%以上。疾病诊断置信度最高可达90%，但仅供参考，最终诊断需要医生判断。"

### Q3: "如何保证医疗安全？"
**A:** "系统有三重保障：
1. 明确标注'仅供参考，不作为诊断依据'
2. 自动识别紧急情况并提示立即就医
3. 药物相互作用和禁忌症检查"

### Q4: "和现有产品有什么区别？"
**A:** "主要创新点是双端设计，同时服务患者和医生。患者端注重易用性和情感关怀，医生端提供专业的辅助诊断和用药决策支持。"

### Q5: "语音识别准确率如何？"
**A:** "使用了Paraformer-Large模型，并针对医疗场景添加了35+个医疗热词，医疗术语识别率显著提升。"

### Q6: "能处理方言吗？"
**A:** "当前版本主要支持普通话。方言支持是下一步计划，可以通过扩充ASR模型的训练数据来实现。"
